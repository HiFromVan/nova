# Nova 架构设计

## 系统定位

```
任何硬件  +  Nova  +  Becoming  =  通用人形机器人
```

- **Becoming**：认知大脑，理解世界，决定做什么
- **Nova**：运动身体，执行动作，驱动硬件
- **硬件**：宇树 H1 / G1 或任意人形机器人

## 频率层级

| 层级 | 频率 | 职责 | 能否缺失 |
|------|------|------|---------|
| Becoming 认知 | 1–10 Hz | 视觉理解、语言、任务规划 | 可以——Nova 有 fallback 步态 |
| Nova 运动规划 | 50 Hz | 步态生成、轨迹规划 | 否 |
| Nova 控制 | 500 Hz | 关节控制、平衡、安全 | 否 |
| 硬件驱动 | 1 kHz | 电机 PWM、传感器采集 | 否（硬件内部） |

**关键原则**：平衡和防摔倒是 Nova 自主的 500Hz 逻辑，不依赖 Becoming。Becoming 断连，机器人仍能站立。

## 两种控制模式

### 高层模式（当前阶段）
```
Becoming 输出语义指令 → Nova 负责运动规划和执行
示例：{"move": {"vx": 0.5, "vy": 0.0, "wz": 0.0}}
```

### 端到端模式（后期 VLA）
```
Becoming 直接输出关节角度数组 → Nova 安全检查后执行
示例：{"action": [0.1, -0.2, 0.5, ...]}  // 19个关节
对齐 LeRobot 接口标准，可直接复用 π0 / ACT / Diffusion Policy
```

## 开发态 vs 生产态

### 开发态（现在）
```
Mac M4 Pro
  ├── Becoming（Python）
  └── Nova（Rust）
        │ gRPC（局域网）
Windows RTX 3090
  └── Isaac Lab（物理仿真）
```
Isaac Lab 仅用于仿真验证，**不是生产架构的一部分**。

### 生产态
```
H1 板载电脑
  ├── Becoming（板载 GPU，推理）
  └── Nova（板载 CPU，500Hz 控制）
        │ DDS（CycloneDDS）
H1 电机控制器（硬件内部，1kHz）

Mac（可选）
  └── 监控 / 遥操作 / 日志
```

## Nova 内部模块

```
nova/src/
  main.rs              入口，组装所有层
  becoming/            接收 Becoming 指令的 gRPC server
  control/
    mod.rs             控制环路（频率调度）
    safety.rs          安全守卫（关节限位、倒地检测）
  robot/
    mod.rs             RobotIO trait（硬件抽象）
    sim.rs             Isaac Lab gRPC（仿真）
    unitree.rs         CycloneDDS（宇树真机）
  brain_interface/
    mod.rs             BrainInterface trait
    baseline.rs        BaselineGait（Becoming 断连时的 fallback）
    types.rs           SensorData / MotorCommands
```

## 与 LeRobot 对齐

| LeRobot 概念 | Nova + Becoming 对应 |
|-------------|---------------------|
| Policy（ACT / π0） | Becoming（VLA 模型） |
| Robot env | Nova（硬件抽象层） |
| Observation space | Nova → Becoming 状态反馈 |
| Action space | 关节角度数组（19维） |

端到端模式的接口直接对齐 LeRobot，Becoming 可零改动复用 LeRobot 的训练 pipeline。
