# Nova 部署架构

## 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                Mac (开发和控制机)                         │
│  ┌──────────────┐         ┌──────────────┐             │
│  │  Becoming    │────────▶│    Nova      │             │
│  │  (决策系统)   │         │  (Rust接口)   │             │
│  └──────────────┘         └──────┬───────┘             │
│                                   │                      │
│                                   │ gRPC (端口 50051)    │
└───────────────────────────────────┼──────────────────────┘
                                    │
                          局域网 (192.168.x.x)
                                    │
┌───────────────────────────────────▼──────────────────────┐
│         Windows (3090 仿真服务器)                         │
│  ┌──────────────────────────────────────────────┐       │
│  │         Isaac Lab Simulator                   │       │
│  │  - NVIDIA 3090 GPU 加速                       │       │
│  │  - gRPC Server (监听 0.0.0.0:50051)          │       │
│  │  - 人形机器人仿真环境                          │       │
│  └──────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────┘
```

## 硬件配置

### Mac (开发机)
- **型号**: Apple M4 Pro
- **用途**: 代码开发、决策系统运行
- **运行**: Nova (Rust) + Becoming (决策)

### Windows (仿真服务器)
- **GPU**: NVIDIA RTX 3090
- **用途**: 物理仿真、渲染
- **运行**: Isaac Lab + gRPC Server

## 通信协议

### gRPC
- **端口**: 50051
- **协议**: HTTP/2
- **序列化**: Protocol Buffers
- **延迟**: < 10ms (局域网)

### 消息类型

1. **SensorData** (Isaac Lab → Nova)
   - 位置、速度、姿态
   - 关节角度、速度
   - 足部接触状态

2. **MotorCommands** (Nova → Isaac Lab)
   - 关节目标角度
   - 关节力矩（可选）
   - 期望速度

## 数据流

### 控制循环 (60 Hz)
```
1. Nova 发送 MotorCommands → Windows
2. Isaac Lab 执行物理仿真 (1/60 秒)
3. Isaac Lab 返回 SensorData → Mac
4. Becoming 决策
5. 重复
```

### 延迟分析
- 网络延迟: ~2ms (局域网)
- 仿真步长: ~16ms (60 Hz)
- 决策时间: ~5ms
- **总延迟**: ~23ms (43 Hz)

## 网络配置

### Windows 端
1. 获取 IP 地址: `ipconfig`
2. 开放防火墙端口 50051
3. 启动 Isaac Lab 服务器

### Mac 端
1. 配置服务器地址: `192.168.x.x:50051`
2. 启动 Nova 客户端
3. 连接测试

## 开发工作流

### 1. 启动仿真服务器 (Windows)
```bash
cd isaac_env
python server.py
```

### 2. 启动 Nova (Mac)
```bash
cd nova
cargo run
```

### 3. 测试连接
```bash
# Mac 上测试
grpcurl -plaintext 192.168.x.x:50051 list
```

## 故障排查

### 连接失败
1. 检查 Windows 防火墙
2. 检查网络连通性: `ping 192.168.x.x`
3. 检查端口占用: `netstat -an | grep 50051`

### 延迟过高
1. 使用有线网络（不要用 WiFi）
2. 关闭其他网络应用
3. 检查 CPU/GPU 负载

### 仿真不稳定
1. 降低仿真频率
2. 检查 GPU 温度
3. 更新 NVIDIA 驱动
