# Becoming 设计文档

## 定位

Becoming 是 Nova 的认知大脑。Nova 负责"怎么动"，Becoming 负责"做什么"。

```
现实世界 → 摄像头/麦克风 → Becoming → 语义指令 → Nova → 电机
```

---

## 模型架构：VLA（视觉-语言-动作模型）

```
┌─────────────────────────────────────────────────────┐
│                   Becoming                           │
│                                                      │
│  输入                                                │
│  ├── RGB 摄像头（头部 + 手腕）                        │
│  ├── 深度摄像头（环境感知）                            │
│  ├── 麦克风（语音指令）                               │
│  └── Nova 状态反馈（关节角度、位置、是否稳定）          │
│                                                      │
│  ┌──────────────────────────────────────────────┐   │
│  │  视觉编码器（Vision Encoder）                  │   │
│  │  SigLIP / DINOv2 → 图像 token                │   │
│  └──────────────────────┬───────────────────────┘   │
│                         │                            │
│  ┌──────────────────────▼───────────────────────┐   │
│  │  语言模型骨干（LLM Backbone）                  │   │
│  │  Qwen2.5 / LLaMA3 / Gemma2                   │   │
│  │  理解指令 + 推理 + 规划                        │   │
│  └──────────────────────┬───────────────────────┘   │
│                         │                            │
│  ┌──────────────────────▼───────────────────────┐   │
│  │  动作头（Action Head）                         │   │
│  │  高层模式：输出语义指令                         │   │
│  │  端到端模式：输出关节角度数组（19维）            │   │
│  └──────────────────────────────────────────────┘   │
│                                                      │
│  频率：1–10 Hz（受限于大模型推理速度）                 │
└─────────────────────────────────────────────────────┘
```

## 参考架构（业界）

| 项目 | 机构 | 特点 |
|------|------|------|
| π0 | Physical Intelligence | Flow matching + VLM，端到端 |
| RT-2 | Google DeepMind | VLM 直接输出动作 token |
| OpenVLA | 斯坦福 | 开源 VLA，基于 LLaVA |
| ACT | 斯坦福 | Transformer，模仿学习 |
| Diffusion Policy | CMU | 扩散模型生成动作序列 |

Becoming 的路径：**先高层模式（LLM + 语义指令）跑通，后期演进到端到端 VLA**。

## 输出接口（发给 Nova）

```json
// 移动
{"move": {"vx": 0.5, "vy": 0.0, "wz": 0.0}}
```

## 开发阶段规划

### 阶段一（现在）：键盘代替 Becoming
- Becoming 不参与，键盘直接控制 Nova
- 验证 Nova 控制逻辑和硬件接入

### 阶段二：LLM + 高层指令
- Becoming 接入 Claude / GPT API
- 语音/文字 → LLM 解析 → 语义指令 → Nova
- 通信：gRPC（本机）

### 阶段三：接入视觉
- 接入摄像头，VLM 理解场景
- "把桌上的杯子拿过来" → 视觉定位 + 运动规划

### 阶段四：端到端 VLA
- 训练或微调 VLA 模型
- 直接输出关节角度，对齐 LeRobot
- 收集遥操作数据 → 训练 → 部署

## 运行位置

| 阶段 | 运行位置 | 硬件要求 |
|------|---------|---------|
| 开发阶段 | Mac M4 Pro | Apple Neural Engine，够跑 7B 模型推理 |
| 生产阶段 | H1 板载 GPU | NVIDIA Orin 或同等算力 |

**重要**：生产阶段 Becoming 必须跑在机器人板载计算上，不能依赖外部网络。
