# Nova 常见问题解答

> 软件背景开发者在设计人形机器人系统时的核心疑问

---

## Q1：仿真训练和策略网络是干什么的？

训练一个神经网络模型让机器人做决策。

传统控制是手写规则：
```
if 左脚离地 and 重心偏右:
    右腿发力 X 牛顿
```
几千行 if-else，换个地形就挂。

强化学习训练的模型：
```
输入：传感器数据（关节角度、IMU、足底力）
输出：各关节目标角度
// 经过几亿次仿真试错，学会了在各种情况下怎么走路
```

训练过程全在仿真里完成，训练完把模型部署到真机，每 5ms 推理一次。

## Q2：宇树/Tesla 这些公司用什么技术栈？

| 阶段 | 工具 | 语言 |
|------|------|------|
| 仿真训练 | Isaac Lab / MuJoCo | Python |
| 策略网络 | PyTorch / JAX | Python |
| 实时部署 | 自研运行时 | C++ |
| 硬件驱动 | 厂商 SDK | C++ |

宇树不用 ROS2，用自研的基于 Cyclone DDS 的通信中间件，比 ROS2 轻，实时性更好。

---

## Q3：Nova 用 Rust 合适吗？需要 ROS2 吗？

Rust 完全合适，实时控制和硬件驱动 Rust 是正确选择，性能接近 C++ 且内存安全。

不需要 ROS2。ROS2 是科研和原型阶段的工具，量产产品基本都自研或绕过。Nova 自己定义接口标准更轻量更可控。

---

## Q4：Isaac Lab 是什么？需不需要用？

Isaac Lab 是 NVIDIA 出的独立仿真软件，用 GPU 同时跑几千个机器人并行训练运动策略模型。

**现阶段不需要。** 原因：

- Isaac Lab 是用来训练"机器人怎么走路"的运动策略模型
- 宇树 G1 买来就会走路，运动控制出厂自带，不需要重新训练
- becoming 的定位是高层认知决策，不是运动策略

只有当你需要支持自研硬件、或者训练端到端 VLA 模型时才需要它，那是很后期的事。

---

## Q5：专用机器人（宇树）和通用机器人（Tesla）架构有什么区别？

**专用（宇树/波士顿动力）**
```
训练目标：把特定动作做好
模型：运动策略模型
输入：传感器数据
输出：关节角度
```

**通用（Tesla Optimus/Figure）**
```
训练目标：理解世界、完成任意任务
模型：视觉语言行动模型（VLA）
输入：摄像头画面 + 语言指令
输出：动作序列
```

核心差异：通用机器人的大脑是多模态大模型，不是运动策略网络。Tesla 的核心是端到端：摄像头画面进去，关节角度出来，中间是一个大模型，需要大量人类遥操作数据训练。

---

## Q6：becoming 需要直接接硬件吗？

不需要。职责划分：

```
becoming 接：摄像头画面、麦克风、Nova 反馈的机器人状态
Nova 接：电机驱动器、IMU、力传感器

becoming 永远不直接控制电机
它只告诉 Nova "做什么"，Nova 决定"怎么做"
```

类比：becoming 是大脑，Nova 是小脑+脊髓。你走路不需要大脑思考怎么控制每块肌肉。

---

## Q7：完整的数据流是什么样的？

```
现实世界
  ↓ 摄像头、麦克风
becoming
  看到什么、听到什么 → 决定做什么
  ↓ 结构化指令（"走到桌子旁边"、"拿起杯子"）
Nova
  把指令转成运动执行
  ↓ 关节角度/力矩
硬件电机
  ↓
机器人动起来
  ↓ 传感器数据
Nova 采集 → 反馈给 becoming（闭环）
```

Nova 内部也有自主控制：平衡、防摔倒、步态执行，这些不需要 becoming 参与。

## Q8：长期会不会需要 GPU 和 Isaac Lab 训练？

会，但目的不同，时间也不同：

| 阶段 | 是否需要 | 用途 |
|------|---------|------|
| 现在 | 否 | - |
| becoming 做视觉理解 | 需要 GPU 推理 | 跑视觉/语言模型（推理不是训练） |
| becoming 做端到端控制 | 需要 GPU 训练 | 训练 VLA 模型 |
| 支持自研硬件 | 需要 GPU 训练 | 训练运动策略 |

推理和训练是两回事。跑 LLM/视觉模型需要 GPU，但推理用消费级显卡就够。

**结论：现阶段 Rapier 仿真 + Rust 够用。中期接宇树真机，运动控制用宇树自带的。长期 becoming 需要 GPU 跑视觉语言模型。**

---

## Q9：Nova 和 becoming 怎么通信？

分阶段演进：

**第一阶段（现在）**：不通信，键盘代替 becoming 调试
**第二阶段**：TCP + JSON，最简单，先跑通
```json
{"command": "move", "direction": "forward", "speed": 0.5}
```
**后期**：换 gRPC + protobuf，语言无关，接口规范

## Q10：Nova 的商业价值是什么？

```
任何硬件  +  Nova  +  becoming  =  通用机器人
```

Nova 是硬件和大脑之间的标准化中间层：
- 接入的硬件品牌越多，生态越强
- becoming 的数据越多，能力越强
- 两者相互增强

类比：Nova 是 Android（硬件适配层），becoming 是 Google 服务（智能能力）。目标客户是买了机器人硬件但没有软件能力的企业。
